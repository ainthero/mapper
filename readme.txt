Описание формата
Для сериализации используется формат, очень похожий на Json.
Состоит из неупорядоченного множества пар ключ-значение.
Ключ всегда является строкой, например "key", "0"
Значение является либо строкой, либо новым множеством пар ключ-значение, указанным в фигурных скобках.
После каждой пары ключ значение пишется запятая, если пара не последняя в записи.
Пробелы и переносы строк запрещены.
